## 测试场景说明

### 权限测试场景
1. **管理员权限**: 可以测试组织管理、成员管理等高级功能
2. **普通成员权限**: 可以测试基本的组织功能
3. **待审核权限**: 可以测试权限申请和审核流程

### 状态测试场景
1. **正常状态**: 可以正常使用所有功能
2. **待审核状态**: 功能受限，需要审核通过
3. **暂停状态**: 账户被暂停，无法正常使用

### 组织类型测试场景
1. **企业组织**: 阿里巴巴、腾讯、字节跳动
2. **学校组织**: 北京大学、清华大学

### 跨组织测试场景
- 可以测试不同组织间的用户交互
- 可以测试组织间的项目协作
- 可以测试跨组织的权限控制

## 使用说明

1. **登录测试**: 使用上述用户名和密码进行登录测试
2. **权限测试**: 使用不同权限的用户测试功能访问控制
3. **状态测试**: 使用不同状态的用户测试业务流程
4. **组织测试**: 使用不同组织的用户测试组织相关功能

## 清理测试数据

如需清理测试数据，可以运行以下命令：

```bash
python manage.py create_test_users --clear
```

## 注意事项

1. 这些用户仅用于测试目的，请勿在生产环境中使用
2. 所有测试用户的密码都是 `test123456`，请注意安全
3. 测试完成后建议清理测试数据
4. 如需修改测试数据，可以编辑 `user/management/commands/create_test_users.py` 文件
